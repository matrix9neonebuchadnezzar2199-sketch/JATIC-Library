SQLite スレッドエラー修正（GUI 起動時 DL）

2026-05-20 incident tool ok

背景

GUI 起動後の「今すぐ更新確認」で SQLite objects created in a thread can only be used in that same thread が発生。

原因

AsyncTaskWorker（QThread）内の StartupScheduler が、メインスレッドで開いた self._repo を参照していた。

対応

DL / 再 DL はワーカー内で Repository(DB_PATH) を別接続。PRAGMA WAL + check_same_thread=False は保険。test_repository_threading.py 追加。
