## 2. 操作追跡（主要フロー）

### 2.1 保管庫ソート変更

`LibraryTab._on_sort_changed` → メモリ上 `library_default_sort` 更新 → `refresh()` → 300ms デバウンス → `sort_changed` → `MainWindow._on_library_sort_changed` → **dirty なら** `ConfigStore.patch_library_sort`、否则 `save`。

### 2.2 設定保存 / dirty

ウィジェット変更 → `_mark_dirty` → タイトル `*`。保存時のみ `apply_to_config` + `ConfigStore.save` + `config_saved`。

### 2.3 今すぐ更新確認

`run_update_check`: Chromium ガード → dirty ブロック → **ワーカー二重 / save_root 未設定をダイアログ表示前に判定** → 進捗ダイアログ → `AsyncTaskWorker` → `StartupScheduler.run_check`。

### 2.4 月次エクスポート

コンテキストメニューで ZIP / CSV を **別シグナル** → `_on_export_month_zip` / `_on_export_month_csv`（Yes/No 二重確認を廃止）。

### 2.5 ファイル削除

`delete_requested` → DB 行削除 + **tag_assignments 削除** → manifest → `unlink`。

## 3. 発見した問題と対応

| 優先 | 問題 | 原因 | 対応 |
|---|---|---|---|
| High | dirty 中のソート保存で設定が上書き | `_on_library_sort_changed` が常に `save(_config)` | `patch_library_sort` + dirty 分岐 |
| High | メニュー「テーマ」で dirty 喪失 | `_set_theme` が `save` + `load_from_config()` | テーマのみ反映、`sync_theme_widget`、dirty 時は save しない |
| Medium | スキップ後 `last_check_at` が再起動で戻る | `_touch_last_check(persist=False)` | skip/error でも `persist=True` |
| Medium | 更新確認で空の進捗ダイアログ | ガード順序 | ワーカー / save_root チェックを `show()` 前へ |
| Low | 再 DL が save_root 未設定で無言 return | ガード欠如 | QMessageBox + 設定タブへ |
| Low | ZIP/CSV メニューが同一ハンドラ | 単一 `export_month_requested` | 2 シグナルに分割 |
| Low | 削除後タグ割当が DB に残る | `delete_tag_assignments` 未呼び出し | 削除処理に追加 |
