# 性能計測ルール（INST_32 以降）

性能関連の受け入れ条件（表示時間・RSS ピーク等）を含む実装指示書に共通で適用する。正本は本ファイル。進捗の併記は [DEV_STATUS.md](../DEV_STATUS.md)。

---

## 計測ルール

実装コミットと計測は **別タイミングで構ってよい**。実装完了時点で `master` へのマージ可とする。ただし以下を必須とする。

1. 性能関連の受け入れ条件を持つ指示書は、計測完了まで **「計測待ち」** ステータスとし、`docs/DEV_STATUS.md` にその旨を明記する。
2. `v0.1.0-beta.X` 等の **リリースタグ付与**は、対象指示書の計測がすべて **「達成」** となった後にのみ実施する。
3. 計測 **「未達」** と判定された場合、後続指示書（INST_33 等）でチューニングを行い、再計測で「達成」となるまでリリースタグを保留する。
4. PR 本文・完了報告・計測記録コミットには **「計測未済」「達成（数値）」「未達（数値）」** のいずれかを必ず記載する（数値のみで判定を曖昧にしない）。

### 守るべき本質

- **守る:** 未達のまま本番リリース（beta タグ・配布物の品質ゲート）に進まないこと。
- **手段として不要:** 未達・未測定の実装コミットを revert してトランクを空けること（チューニングは追加コミットで収束する）。

## ラボ計測の補足（参考）

### 保管庫タブ（表示時間）

1. `%LOCALAPPDATA%\JATIC-Library\library_scan_cache.json` を削除（初回計測）。
2. アプリ起動 → 保管庫タブ。`LibraryTab.__init__` 〜 `_rebuild_tree()` 完了直後を `time.perf_counter()` で計測（一時ログ可、**計測用コードはコミットしない**）。
3. 終了 → 再起動 → 同タブ（キャッシュ温）。

### 統合 CSV（RSS ピーク）

51 地域相当で月次 CSV エクスポート（または postprocess 相当）実行中のピーク。

- **PowerShell（`psutil` 不要）:** `Get-Process -Name python | Select-Object WorkingSet64, PeakWorkingSet64` を 1 秒間隔でループ。
- **目視:** `resmon.exe` → メモリ → 該当プロセスのコミット / ワーキングセットピーク。
